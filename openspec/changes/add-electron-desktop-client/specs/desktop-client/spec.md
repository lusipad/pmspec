# desktop-client 规格增量

## ADDED Requirements

### Requirement: 桌面客户端基础启动
系统 MUST 提供基于 Electron 的桌面客户端入口，并在启动后展示主界面。

#### Scenario: 开发模式启动桌面客户端
- **WHEN** 开发者执行桌面端开发脚本
- **THEN** 系统启动 Electron 主进程并创建主窗口
- **THEN** 主窗口加载本地开发服务器地址并显示界面内容

#### Scenario: 生产模式启动桌面客户端
- **WHEN** 用户启动已安装的桌面应用
- **THEN** 系统从本地构建资源加载渲染界面
- **THEN** 主窗口在无外部开发服务前提下可正常打开

### Requirement: 主进程窗口生命周期管理
系统 MUST 在主进程中管理窗口创建、激活与退出行为，确保桌面端运行稳定。

#### Scenario: 首次启动创建主窗口
- **WHEN** 应用触发 ready 生命周期事件
- **THEN** 系统创建单一主窗口并绑定预加载脚本

#### Scenario: 窗口关闭后的应用行为
- **WHEN** 用户关闭全部窗口
- **THEN** 系统按平台约定执行退出或等待激活逻辑
- **THEN** 再次激活应用时可重新创建主窗口

### Requirement: 预加载安全桥接能力
系统 MUST 通过预加载层向渲染进程暴露最小必要 API，并隔离 Node.js 运行时能力。

#### Scenario: 渲染进程调用受控 API
- **WHEN** 渲染进程调用预加载暴露的白名单 API
- **THEN** 调用通过 IPC 转发到主进程并返回结果
- **THEN** 未在白名单中的通道不可被调用

#### Scenario: 默认安全隔离生效
- **WHEN** 渲染进程运行在主窗口中
- **THEN** 页面无法直接访问 Node.js 全局对象
- **THEN** 页面仅能访问预加载层显式暴露的接口

### Requirement: 桌面应用打包与分发产物
系统 MUST 提供可重复执行的桌面应用打包流程，并生成可安装分发的构建产物。

#### Scenario: 执行打包脚本生成安装包
- **WHEN** 开发者执行桌面端打包脚本
- **THEN** 系统先完成渲染层与桌面进程构建
- **THEN** 系统输出目标平台安装包到约定目录

#### Scenario: 打包产物可安装并启动
- **WHEN** 用户安装生成的桌面端安装包
- **THEN** 应用可完成安装并成功启动
- **THEN** 启动后可加载主界面并执行基础交互
