# PMSpec Refine

细化和优化现有的项目结构，包括分解大任务、合并小任务、优化依赖关系等。

## 使用方法

提供需要优化的项目 ID 或整体优化建议。

[需要优化的项目ID或描述]

## 输出格式

```markdown
# 项目优化建议

## 当前结构分析
- Epic 数量: [数量]
- Feature 数量: [数量]
- User Story 数量: [数量]
- 平均 Story 工时: [工时]h
- 最大 Story 工时: [工时]h
- 最小 Story 工时: [工时]h

## 优化建议

### 1. 任务分解
**需要分解的大任务:**
- FEAT-XXX: [Feature 标题] ([当前工时]h → 分解为多个小任务)
  - 建议分解:
    - STORY-XXX: [子任务1] ([工时]h)
    - STORY-XXX: [子任务2] ([工时]h)

### 2. 任务合并
**可以合并的小任务:**
- STORY-XXX, STORY-XXX: [相关描述] → 合并为新的 STORY-XXX ([合并工时]h)

### 3. 依赖优化
**依赖关系调整:**
- FEAT-XXX: [当前依赖] → [建议调整]
- 调整理由: [详细说明]

### 4. 技能分配优化
**技能匹配建议:**
- FEAT-XXX: [当前技能] → [建议技能] ([负责人])
- STORY-XXX: [技能调整建议]

## 优化后的结构

### Epic: [Epic 标题]
- Features: [数量] (总计: [工时]h)
  - FEAT-XXX: [Feature 标题] ([工时]h)
    - Stories: [数量]
    - 技能要求: [技能列表]

## 实施计划
1. [步骤1]: [具体操作]
2. [步骤2]: [具体操作]
3. [步骤3]: [具体操作]

## 预期效果
- 任务粒度更加均匀
- 技能匹配更合理
- 依赖关系更清晰
- 估算准确性提升 [百分比]%
```

## 优化原则

1. **任务粒度**: User Story 应该在 1-3 天内可完成
2. **技能匹配**: 充分利用团队成员的技能优势
3. **依赖管理**: 减少关键路径上的阻塞点
4. **风险控制**: 识别和缓解高风险任务
5. **可追踪性**: 确保每个任务都有明确的验收标准
